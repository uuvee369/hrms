# HRMS Batch Import Tool

เครื่องมือช่วยนำเข้าข้อมูลพนักงานและสร้างบัญชีผู้ใช้งานเข้าระบบ HRMS (Healthcare Risk Management System - ระบบสารสนเทศการบริหารจัดการความเสี่ยงของสถานพยาบาล) แบบอัตโนมัติรวดเดียวผ่านไฟล์ Excel

##  ระบบนี้ถูกสร้างมาเพื่อแก้ปัญหาอะไร?

** ปัญหาของระบบ HRMS เดิม (Pain Points)**
1. **ขั้นตอนซ้ำซ้อนและทำงานหลายหน้าจอ:** การเพิ่มพนักงาน 1 คน ฝ่าย HR ต้องทำงาน 2 ส่วนหลัก คือ 1) เข้าไปหน้าแรกเพื่อ "บันทึกชื่อบุคลากร" ลงระบบ จากนั้น 2) ต้องสลับไปเปิดหน้าใหม่เพื่อ "สร้างบัญชีผู้ใช้" โดยในหน้านี้จะต้องทำการกรอก Username, กำหนดรหัสผ่าน, ค้นหาชื่อพนักงานที่เพิ่งสร้างเพื่อนำมาแมปคู่กัน, และแมปสิทธิ์กลุ่มผู้ใช้งาน ซึ่งทั้งหมดนี้ต้องทำด้วยมือทีละคนและสลับหน้าไปมา ทำให้เสียเวลามาก
2. **พิมพ์ผิดบ่อยและตกหล่น:** การต้องกรอกข้อมูลด้วยมือ (Manual Entry) จำนวนมากๆ ทำให้มีโอกาสพิมพ์ผิด หรือลืมผูกบัญชีผู้ใช้งาน

** การแก้ปัญหา (Solutions)**
1. **นำเข้าแบบ One-Stop Service:** ระบบนี้รวบขั้นตอนทุกอย่างไว้ในหน้าเดียว แค่ฝ่าย HR เตรียมไฟล์ Excel ที่มีคอลัมน์ชื่อและ Username โยนใส่ระบบปุ่มเดียว ระบบจะวิ่งทำงานแทนคนให้ทั้งหมด!

---

##  ฟีเจอร์หลักและการทำงาน

-  **2-Phase Batch Automation:** เมื่ออัพโหลด Excel ระบบจะฉลาดพอที่จะทำงาน 2 จังหวะให้เอง:
  1. สร้างรายชื่อพนักงานเข้าฐานข้อมูลก่อน
  2. ดึงข้อมูลรหัสพนักงานใหม่ล่าสุดที่เพิ่งสร้าง กลับมาผูกกับ Username / Password เพื่อสร้างบัญชีให้ใช้งานได้ทันที
-  **ระบบป้องกันข้อมูลซ้ำ (Smart Duplicate Detection):** ถ้ารายชื่อใน Excel มีคนที่เป็นพนักงานอยู่แล้ว ระบบจะแสดงสถานะให้รู้ และข้ามการสร้างชื่อซ้ำให้เอง (ทำแค่สร้าง Username ให้แทนถ้ายังไม่มี)
-  **ลบพนักงานแบบรายคน (Single Delete):** มีระบบหน้าจอสำหรับค้นหาและลบพนักงานออก พร้อมแสดงข้อมูลครบถ้วนว่าใครอยู่กลุ่มไหน
-  **ชะลอการส่งมอบข้อมูล (Rate Limiting):** ระบบจะหน่วงเวลา 1 วินาทีในการเซฟทีละคน เพื่อป้องกันไม่ให้เซิร์ฟเวอร์ HRMS ล่มจากการยิงข้อมูลพร้อมกันเยอะๆ

##  วิธีใช้งานเบื้องต้น

### ความต้องการของระบบ
- **Python 3.8+** (สามารถรันโค้ดได้เลย ไม่ต้องใช้คำสั่ง `pip install` ใดๆ เพราะระบบจะติดตั้ง dependencies ที่จำเป็นให้ตอนรันครั้งแรก)

### การเริ่มต้นรันโปรแกรม
```bash
# 1. เปิด Terminal ไปที่โฟลเดอร์โปรเจค
cd d:\hrms

# 2. รัน Server
python server.py

# 3. เปิดเบราว์เซอร์ใช้งานผ่าน URL 
# http://localhost:5000
```
> **การเข้าสู่ระบบ:** ให้ใช้ Username และ Password ของระบบ HRMS เดิม (https://ntw.thai-nrls.org/) ได้เลย

## 📥 การเตรียมไฟล์ Excel

ไฟล์ Excel (รองรับ .xlsx, .xls, .csv) ต้องมีคอลัมน์ดังนี้ (ตัวเล็ก/ใหญ่ หรือมีช่องว่างก็อ่านได้):

| คอลัมน์ที่ระบบค้นหา | ความจำเป็น | คำอธิบาย |
|---------------------|------------|----------|
| `name` หรือ `ชื่อพนักงาน` | ✅ **บังคับ** | ชื่อพนักงานที่ต้องการเพิ่มเข้าระบบ |
| `username` หรือ `ชื่อผู้ใช้` | ❌ ทางเลือก | หากใส่มา ระบบจะสร้างบัญชีผู้ใช้งาน + ตั้งรหัสผ่านที่หน้าเว็บกำหนดให้ด้วย |

*Note: แถวไหนที่ไม่ได้ใส่ Username มา ระบบก็จะสร้างให้แค่เฉพาะรายชื่อพนักงานอย่างเดียว*

## 📁 โครงสร้างไฟล์โปรเจค (Project Structure)

```
hrms/
├── server.py      # Python Proxy Server (สะพานเชื่อม API ของ HRMS)
├── index.html     # หน้าเว็บหลัก (HTML UI)
├── style.css      # Custom CSS เสริมความสวยงามของ Bootstrap
├── app.js         # Core Logic จัดการ State และควบคุมภาพรวม
├── login.js       # จัดการการ Login / Logout
├── upload.js      # จัดการลากวางไฟล์ Excel และแปลงข้อมูล (SheetJS)
├── modes.js       # โค้ดพระเอก! จัดการระบบ 2-Phase นำเข้าข้อมูล
├── delete.js      # โค้ดสำหรับหน้าลบข้อมูลพนักงาน
└── AGENTS.md      # 🤖 คู่มือระดับเจาะลึก (Architecture & API) สำหรับ AI อ่านเวลาซ่อมโค้ด
```
